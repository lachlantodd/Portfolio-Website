Comparathon

Compare two Twitter users head to head.

This Node.js application was created as the final submission to QUT's unit CAB432 Cloud Computing. The requirement was to create a stateless cloud application which utilised cloud scaling and persistence. Since this was a group assignment, I developed it with another student, Tom Hunter.

The web app allows you to enter two Twitter usernames to compare head to head. The 100 most recent (Twitter API limitation) tweets are then grabbed and analysed for sentiment using natural language processing. The QuickChart API is then used to create a stacked graph comparing the two users for positive, negative and neutral tweet sentiment. Finally the chart, as well as the compiled list of all tweets returned from Twitter, AWS S3 and Redis cache.

To bypass the Twitter API 100 tweet and seven day history limitations, an AWS S3 bucket and Elasticache Redis cache is used for persistence. The application was also imaged and its scaling out and back in ability was tested on AWS.
