The Plane Game

C# mobile application created using Unity

The Plane Game was my method of learning to code using C#. I worked on-and-off for about one year to complete it, thanks to my decision to challenge myself and create all code, artwork and music by myself. This also taught me first-hand the value of defining and constraining the scope of a project early. Since I did not do this, I experienced a significant amount of scope creep as I relied on coming up with ideas over time as I developed the concept. By failing to complete this important step, I learnt a valuable lesson to carry with me throughout my professional career.

For this project, I also learnt revision control using Git and GitHub. I did this to prevent a massive loss of data, as well as to prepare to share my source code publically. My project was pushed to GitHub often multiple times per day, after significant progress was made. This came in handy when I ran into out-of-sync branches. I was able to revert my GitHub base to an earlier shared revision and then upload my most recent local changes. The code for this project is publically available on my GitHub.

The Plane Game was also intended to be a tool to test the mobile app store market. I aimed to test the viability of setting up a side business releasing mobile games and applications. I made the game completely free, with no microtransactions or advertisements to give it the best chance at success. What I discovered was that the app stores are quite flooded with choice for the consumer. This meant that without considerable investment in marketing, this was not a viable business strategy. With a small investment, I gained valuable market insight.
